# Challenge 08 — Capture the Output

HQ wants a situation report saved as `~/report.txt`. Two lines, in
this exact order:

- **Line 1**: your operative codename on this box. The system has a
  built-in command that prints who you are.
- **Line 2**: the name of the kernel running this host. One of the
  system-info commands prints kernel info — you want just the
  kernel name itself, nothing else. (When you find the command,
  check its `--help` to see which flag prints only the kernel name.)

The point of this exercise isn't to **type** the answers — anyone
can do that. It's to learn how to **capture a command's output
straight into a file**, so the report is generated by the system
and not by your typing. There are two redirection operators that do
this; one overwrites whatever's in the file, the other appends a
new line to the end. You'll need both.

## Objective

Build `~/report.txt` so that line 1 is your username and line 2 is
the kernel name (`Linux`). No extra lines. Then submit.

## Submit

```
ctf submit 08
```

No flag needed — the checker reads the file and validates it
against the rules above. If something's off it'll tell you which
line is wrong.

## Stuck?

```
ctf hint 08
```
